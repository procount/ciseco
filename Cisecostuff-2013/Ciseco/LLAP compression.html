LLAP compression

Whilst on the outside LAPP looks to be remarkably human in it's 12 character(byte) appearance there is however a "slimmer" way to send this "over the air" (if required). We do this for example with Infra Red because of the slow speeds attainable. For LIN and ANT where an allowable single data payload is limited to 8 bytes this also brings dividends as we dont have to fragment messages.  The "compression" we suggest should be invisable to the user and the normal 12 bytes in/out be compressed/decompressed without thier input. Lets see how we do it.

8 bytes is 64 bits, 12 is 96 so we need to lose 32 bits along the way, here's how.........

Drop the first byte "a", as we know what both ends of the "transmission medium" are there's no need for a generic start byte. The payload part of LLAP (last 9 characters) uses the lower part of ASCII (0-9 & A-Z) so we can drop 2 bits from each of the 9 payload bytes, This then leaves us with the device ID being 2 x 5 bits.
